# Prompt 与验收

仅在复杂生成/编辑、重要文字、多参考图、交互标记、透明背景或需要迭代的任务中阅读本文。模型、尺寸、组图、联网和 CLI 参数分别以 `lite.md`、`pro.md`、`cli.md` 为准。

## 编写原则

- Prompt 默认跟随用户主要输入文本的语言；混合或不明确时沿用当前对话及用户的全局语言习惯。中文、英文都可，不要仅为选择语言进行额外分析、翻译或询问；逐字文字始终保持用户原文。
- 用简洁自然语言表达：场景/背景 → 主体与动作 → 关键细节 → 风格与构图 → 约束与用途。
- 用户描述已经具体时，只整理为清晰规格；不要擅自添加角色、道具、品牌、品牌色、标语或故事。
- 请求较泛时，仅补充能实质提高结果的构图、画幅、用途、光影或材质信息。
- 明确画幅、视角和留白；没有页面或用户依据时，不要强行指定左右布局。
- 删除重复和低优先级要求，避免过长 prompt 分散注意力。

## 分类与结构

生成分类：`photorealistic-natural`、`product-mockup`、`ui-mockup`、`infographic-diagram`、`scientific-educational`、`ads-marketing`、`productivity-visual`、`logo-brand`、`illustration-story`、`stylized-concept`、`historical-scene`。

编辑分类：`text-localization`、`identity-preserve`、`precise-object-edit`、`lighting-weather`、`background-extraction`、`style-transfer`、`compositing`、`sketch-to-render`。

按任务取用以下字段：

```text
用途：<资产将用于何处>
任务类型：<分类 slug>
核心请求：<主要目标>
输入图片：<图一：角色；图二：角色>（可选）
场景/背景：<环境>
主体：<主要对象和动作>
风格/媒介：<摄影、插画、3D 等>
构图/景别：<画幅、视角、焦点、留白>
光线/氛围：<光源、时间、情绪>
色彩/材质：<必要的颜色和纹理>
文字（逐字）："<准确文字>"
必须保持：<不变项>
禁止：<负面约束>
```

不要把模板所有字段或所有限制堆进一个 prompt；短而无歧义优先。

## 文字

- 不需要文字时明确禁止文字、字母、数字、标签、Logo 和水印。
- 需要文字时用引号标出逐字内容，并指定层级、位置、字体气质、字号关系和颜色；不得让模型补充文案。
- 生僻词或易错词可逐字拼写；信息图、表格、坐标轴和密集小字尤其需要逐字验收。
- 准确性重要且反复失败时，改用确定性排版后处理。

## 输入图片与多参考图

- 按 CLI `--image` 顺序称为图一、图二……，并说明每张是编辑目标、内容来源、风格参考或交互标记图。
- 仅作风格、构图或氛围参考的图片属于生成参考，不应被当成编辑目标。
- 合成时描述交互方式，例如「将图二人物置于图一右侧」，并要求匹配比例、透视、遮挡、环境光与接触阴影。
- 需保留图一时，写清身份、姿势、背景、镜头、构图和未编辑区域等不变项。

## 联网搜索

- 用户或 prompt 明确要求联网、搜索、检索最新/实时信息时，保留这项意图并为 Lite 调用加入 `--web-search`。
- 用户未明确要求时，凡是带有具体近期日期的世界局势/时局，以及天气、新闻、行情、赛程、现任人物、近期产品状态等时效事实会实质影响画面内容时，都必须视为联网需求；普通创意生成默认不开启。
- 未指定模型但需要联网时直接选择 Lite；同时明确要求 Pro 或 Pro 专属能力时，先询问用户选择 `Lite 联网` 或 `Pro 生图能力（无模型原生联网）`。
- 启用搜索只提供工具，模型仍会自行决定是否实际搜索；生成后必须核对日期、数字、标题和逐字文字。

## 编辑与标记

- 编辑使用明确句式：`只改 X；保持 Y 不变。`
- `text-localization`：只换文字，保持布局、字体、字号、颜色、对齐、间距和层级。
- `identity-preserve`：锁定面部、体型、姿势、发型、表情和身份，只改指定元素。
- `precise-object-edit`：精确说明对象和位置，保留周围纹理、结构、光线与阴影。
- `lighting-weather`：只改变光照、季节或天气，保留几何、镜头与主体身份。
- `style-transfer`：说明要继承的配色、纹理、笔触或材质，防止复制参考图的具体主体或文字。
- 坐标点、圈选、涂鸦或草图编辑优先选择 Pro。逐一写明标记、原对象、目标对象和位置的对应关系；要求最终移除全部辅助线、点和草图。

## 透明背景

模型不能稳定直接输出透明 alpha。透明 prompt 的唯一可复制模板见 `sample-prompts.md`；色键命令、适用边界、auto-key 失败条件、alpha 契约和三项交付检查见 `chroma-key.md`。不要在此复制第二份模板。

选择键色时检查主体整个色相族而非 exact hex。主体同时包含多种候选键色，或具有毛发、玻璃、烟雾、液体、薄纱、运动模糊、软阴影和半透明结构时停止简单色键方案。

## 验收与迭代

- 实际查看每张图片；检查真实尺寸、格式、主体数量、构图、文字、畸形、重复、参考一致性、编辑不变项和禁止项。
- 自定义 `WIDTHxHEIGHT` 必须精确匹配；尺寸档位则检查是否落在所选模型范围。
- 区分「prompt 要求」和「实际观察」。发现问题后先报告，产生下一次付费请求前取得同意。
- 每次迭代只改一个主要问题，并再次写出关键不变项。

## 分类提示

- `photorealistic-natural`：使用真实摄影语言、自然光和可感知的现实纹理，避免无依据的过度棚拍感。
- `product-mockup`：说明产品、材质、标签清晰度、轮廓与留白；需要文字时逐字要求。
- `ui-mockup`：先说明高保真界面或低保真线框，再定义层级、栅格、模块和设备画幅。
- `infographic-diagram` / `scientific-educational`：定义受众、阅读顺序、标签、箭头和准确性约束；密集文字优先确定性后期。
- `ads-marketing`：像创意 brief 一样给出受众、品牌定位、场景、氛围和确切文案。
- `productivity-visual`：明确是幻灯片、图表或流程图，提供实际标签/数据与层级。
- `logo-brand`：强调简洁轮廓、负空间和可缩放性；如需可编辑标志，应优先使用矢量工作流。
- `historical-scene`：给出地点、年代和必要的服装、道具、环境准确性。
